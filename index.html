Reports



GFlowNet Graph Neighbor Sampling For Link Prediction: Graph Neural Networks (GNNs) are powerful for representation learning but face scalability challenges in handling complex node connectivity. To address this, GFlowNet Graph Neighbor Sampling (GRAPES) was introducted as an adaptive solution that maximizes GNN accuracy within a fixed sampling budget. Our evaluation on link prediction tasks, including negative edge sampling in GCN training, demonstrates promising results, with GRAPES performing competitively or better than state-of-the-art methods on standard datasets (Cora and CiteSeer) across various sample sizes. The code can be accessed here.

Assessing Key Drivers of Birthweight Using R : Aimed to understand and predict Low Birth Weight (LBW) by analyzing a dataset in R. Through rigorous data cleaning, normalization, and advanced regression techniques, including hybrid forward-backward stepwise and ridge regression, key variables influencing birth weight were identified. Factors such as mother's pre-pregnancy weight, birth head circumference, number of prior live births, and parental races emerged as significant, though limitations exist due to unknown population origin in the dataset. The code can be accessed here.

Projects



Neural Language Generation: Fine-Tuning BART for Text Summarization on XSum : Emplpoyed the XSum dataset, which contains short news articles to develop evaluation and optimization of the BART model for text summarization from scratch. Leveraged custom PyTorch functions for cross-entropy loss and ROUGE scoring, along with Greedy Decoding and BEAM Search algorithms. Additionally, fine-tuned hyperparameters for BEAM Search, ultimately achieving a F1 ROUGE score of 43.8. Libraries/Skills Demonstrated: NLP, PyTorch .

Optimizing Urgent Care Services Through Simulation Modeling: Developed a discrete event simulation model in Python to enhance urgent care services. Traced patient interactions and visualized system dynamics by plotting sample paths. Key performance metrics, such as average patient times in the system and waiting times for primary care and mental health consultations, provided insights into operational efficiency. Libraries/Skills Demonstrated: deampy, matplotlib, Discrete Event Simulation, Tracing .

Extracting Patient Data: Using synthetic patient data from the Synthea database that was adopted to fit the OMOP schema, created PySpark scripts to extract relevant patient information. The scripts extract data relating to demographic information, medical measurements, and medical conditions for patients based on recorded visits. Each patient and visit has an unique ID. Included some sample conditions to demonstrate how the scripts work and generated plots to visualize the data. This can be viewed by clicking on the link above. The scripts can be accessed here. Libraries/Skills Demonstrated: Pandas, Seaborn, PySpark.

Water Wars: Using data from the Pacific Institute, conducted analysis on armed conflicts associated with water resources and water systems, as identified from news reports, eyewitness accounts, and other conflict databases. Libraries/Skills Demonstrated: Pandas, Seaborn, GeoPandas, Text Mining.
